# マユミ 资源分析与修复说明

## 结论

这套 `マユミ` 资源里，脸部表情不显示的根因不是 PNG 路径错误，而是 **图层合成顺序错了**。

### 资源规律
- `*.json` 里：
  - `表情` 组包含表情与红晕图层（如 `4478=微笑`, `4477=頬常時`）
  - `制服２武器 / 制服１武器 / 制服２ / 制服１` 是身体底图组
- `png_output/*.png` 与 `layer_id` 一一对应
- 部分 `json` 没有对应 `sinfo`，此时不能依赖 `sinfo` 决定绘制顺序

### 问题原因
在 `マユミＡ_2.json` 里，表情层条目写在身体层条目前面。旧版程序直接按 JSON 中的原始顺序绘制，导致：

1. 先画表情
2. 再画身体底图
3. 身体底图中的“空白脸”把表情盖住

所以预览里会出现：**身体显示了，但脸部表情消失**。

### 修复方案
合成顺序改为固定规则：

1. 固定层
2. 身体底图
3. 表情
4. 红晕

这样即使 JSON 里的原始记录顺序不适合直接绘制，最终预览和导出仍然正确。

## 已同步修改
- 修复 `Project._selected_layers()` 的绘制顺序
- GUI 文字改为：
  - `姿势及其图片分辨率`
  - `身体服装`

## 验证
已用 `マユミ(1).zip` 中的资源复测：
- 旧版：身体有，脸部表情被盖住
- 新版：身体、表情、红晕均正常显示并可导出 PNG
